# Plan: Web UI (`dops open`)

**Spec:** [specs/web-ui.md](../../specs/web-ui.md)
**Status:** IN PROGRESS ‚Äî Phase 1 + Phase 2 complete

## Summary

Add a `dops open` command that serves a Vue 3 web UI for browsing and executing runbooks. Frontend embedded in the Go binary.

## Research Findings

### R1: Vue 3 + Go Embed ‚Äî RESOLVED
- Use `//go:embed all:dist` (the `all:` prefix is critical ‚Äî skips dotfiles without it)
- Directory structure: `web/` (Vue source), `internal/web/dist/` (build output, gitignored)
- SPA fallback: `fs.Stat` check in Go handler ‚Äî if file not found, serve `index.html`
- `fs.Sub(DistFS, "dist")` to strip the prefix for `http.FileServer`
- Real-world examples: Gitea, AdGuard Home, Flipt all use this pattern

### R2: shadcn-vue ‚Äî RESOLVED
- **Production-ready.** v1.x stable, ~9,700 stars, 60+ components, actively maintained
- Backed by Reka UI v2 (headless, accessible)
- All needed components available: Sidebar, Input, Select, Combobox (multi), Dialog, Sonner (toast), Table, Tabs, Badge, Card, Form, Stepper, Resizable
- Components are copy-pasted (not npm dep) ‚Äî full ownership

### R3: TailwindCSS v4 ‚Äî RESOLVED
- CSS-first config via `@theme` directive (no `tailwind.config.js`)
- Use `@tailwindcss/vite` plugin (not PostCSS)
- Auto content detection ‚Äî no `content: [...]` needed
- Replace `tailwindcss-animate` with `tw-animate-css`
- Compatible with shadcn-vue (official migration guide exists)
- Production CSS: ~8-15KB gzipped

### R4: Real-time Streaming ‚Äî RESOLVED: SSE
- SSE over WebSocket: simpler, auto-reconnect, standard HTTP, no firewall issues
- Go implementation: `http.Flusher` interface, ~20 lines
- Cancel via POST endpoint alongside SSE stream
- If future need for bidirectional (stdin), upgrade to WebSocket then

### R5: Build Pipeline ‚Äî RESOLVED: Makefile + gitignored dist
- `make web` builds Vue app into `internal/web/dist/`
- `dist/` gitignored, `dist/.gitkeep` for embed to compile
- Vite dev server + Go API in parallel for development (`vite.config.ts` proxy)
- No `go generate` ‚Äî Makefile is more explicit and avoids stale-asset footgun

### Theme Mapping
dops JSON theme tokens ‚Üí shadcn-vue CSS variables:

| dops Token | CSS Variable |
|---|---|
| `background` | `--background` |
| `backgroundPanel` | `--card`, `--sidebar` |
| `backgroundElement` | `--muted`, `--accent` |
| `text` | `--foreground` |
| `textMuted` | `--muted-foreground` |
| `primary` | `--primary` |
| `border` | `--border` |
| `success` | `--success` (custom) |
| `error` | `--destructive` |

Go serves theme as `<style>:root{...}</style>` block in HTML.

### ANSI Rendering
- **ansi_up**: 4KB, zero dependencies, converts ANSI ‚Üí HTML spans. Start here.
- **terminal-to-html** (Buildkite): Go-side conversion if server-side rendering preferred
- **xterm.js**: Full terminal emulator, only if cursor movement needed (overkill for v1)

### Binary Size Impact
- Vue 3 + Tailwind SPA: ~200KB-1MB uncompressed added to Go binary
- Go binary is ~7MB base, so ~3-15% increase. Negligible.

## Risks

- Largest feature ‚Äî scope creep potential is high. Strict phase gates.
- Node.js build dependency for frontend development (not needed for Go-only contributors)
- Binary size increase ~200KB-1MB (acceptable)
- ANSI rendering edge cases (cursor movement) ‚Äî start simple with ansi_up
- Security: localhost-only for v0.10.0, no auth
