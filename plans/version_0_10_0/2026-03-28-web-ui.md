# Plan: Web UI (`dops open`)

**Spec:** [specs/web-ui.md](../../specs/web-ui.md)
**Status:** TODO — Research phase

## Summary

Add a `dops open` command that serves a Vue 3 web UI for browsing and executing runbooks. Frontend embedded in the Go binary.

## Research Required (Before Implementation)

### R1: VueJS 3 + Go Embed
- [ ] How to embed a pre-built Vue SPA via `//go:embed`
- [ ] Directory structure: `web/` source, `internal/web/dist/` embedded
- [ ] Development workflow: `npm run dev` with proxy to Go API

### R2: shadcn-vue
- [ ] Component maturity assessment (stable? actively maintained?)
- [ ] Available components: sidebar, form inputs, dialog, toast, table
- [ ] Theming: can we map dops JSON themes to CSS variables?

### R3: TailwindCSS v4
- [ ] Build pipeline: does v4 work with Vue 3 + Vite?
- [ ] CSS-first config vs JS config
- [ ] Output size for embedded distribution

### R4: Real-time Streaming
- [ ] SSE vs WebSocket for execution output
- [ ] SSE: simpler, HTTP-native, auto-reconnect — likely better fit
- [ ] WebSocket: bidirectional (needed for Ctrl+C cancel?)
- [ ] Recommendation: SSE for output streaming, POST for cancel

### R5: Build Pipeline
- [ ] Makefile target: `make web` builds Vue app into `internal/web/dist/`
- [ ] `go generate` alternative for CI
- [ ] Contributors who only touch Go: committed `dist/` or CI-only build?
- [ ] Recommendation: commit built assets, `make web` to rebuild

## Implementation Steps (Post-Research)

### Phase 1: API Server
- [ ] Create `cmd/open.go` — Cobra command with `--port` and `--no-browser` flags
- [ ] Create `internal/web/server.go` — HTTP server with graceful shutdown
- [ ] API routes:
  - `GET /api/catalogs` — list catalogs with runbooks
  - `GET /api/runbooks/:id` — runbook detail with parameters
  - `POST /api/runbooks/:id/execute` — execute with params, return execution ID
  - `GET /api/executions/:id/stream` — SSE stream of output
  - `POST /api/executions/:id/cancel` — cancel execution
- [ ] Add API tests

### Phase 2: Vue Frontend Scaffold
- [ ] Initialize Vue 3 + Vite + TypeScript in `web/`
- [ ] Install TailwindCSS v4 + shadcn-vue
- [ ] Create layout: sidebar + main content area
- [ ] Embed built assets in `internal/web/dist/` via `//go:embed`

### Phase 3: Catalog Browser
- [ ] Sidebar component: collapsible catalog tree
- [ ] Runbook card: name, description, risk badge, parameter count
- [ ] Filter/search input

### Phase 4: Execution Flow
- [ ] Parameter form: render all field types from runbook schema
- [ ] Risk confirmation dialog (matching TUI behavior)
- [ ] Execution view: streaming output with ANSI rendering
- [ ] Cancel button wired to `/api/executions/:id/cancel`

### Phase 5: Theme Integration
- [ ] Map dops JSON theme colors to CSS custom properties
- [ ] Theme selector in web UI header
- [ ] Default to configured theme from config.json

## Risks

- Largest feature — scope creep potential is high
- Node.js build dependency for frontend development
- Embedded asset size (Vue + Tailwind) adds to binary size
- ANSI-to-HTML rendering in browser needs a library (e.g., ansi-to-html)
- Security: localhost-only mitigates most concerns, but execution endpoint still powerful
