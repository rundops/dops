## 1. MCP Tool Schema Defaults

**Goal:** Saved vault values appear as `default` in MCP tool JSON Schema.

### Setup
```bash
# Save a parameter value so the vault has something to resolve
dops run default.hello-world --param greeting="Saved-Hello" --param name="World"
```

### Test
```bash
# Query the MCP server for tool schemas via JSON-RPC
(echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}'
echo '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}') | dops mcp serve --transport stdio 2>/dev/null | python3 -m json.tool | grep -A 20 '"default.hello-world"'
```

- [ ] **1a.** `default.hello-world` appears in the tools list
- [ ] **1b.** The `greeting` property contains `"default": "Saved-Hello"`
- [ ] **1c.** Parameters without saved values have no `default` field
- [ ] **1d.** Secret parameters (check `demo.backup-database`) do not appear in the schema properties

## 6. MCP Skills

**Goal:** Skills loaded from disk, exposed as MCP prompts, hidden from TUI.

### Setup

Create a test skill:
```bash
mkdir -p ~/.dops/catalogs/default/k8s-guide
cat > ~/.dops/catalogs/default/k8s-guide/runbook.yaml << 'YAML'
name: Kubernetes Scaling Guide
type: skill
description: Context for AI agents about Kubernetes scaling operations
trigger: scale, resize, replicas
YAML

cat > ~/.dops/catalogs/default/k8s-guide/skill.md << 'MD'
# Kubernetes Scaling

When scaling deployments:
1. Check current replica count with `kubectl get deploy`
2. Use the scale-deployment runbook for the actual operation
3. Verify pods are healthy after scaling

Related runbooks:
- infra.scale-deployment
MD
```
